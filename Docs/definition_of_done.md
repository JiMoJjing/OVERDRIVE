# Definition of Done

## 목적

Definition of Done은 작업을 완료했다고 판단하기 위한 기준입니다.

Codex가 대충 구현하고 끝내지 않도록, 작업 완료 전에 확인할 항목을 명확히 합니다.

---

## 공통 완료 기준

작업 완료 시 해당되는 항목을 확인합니다.

- 코드가 컴파일되었거나, 실행해야 할 컴파일 명령이 명시되어 있습니다.
- 변경 파일이 정리되었습니다.
- 구현 요약이 작성되었습니다.
- 게임플레이 영향이 설명되었습니다.
- 서버 / 클라이언트 책임이 명확합니다.
- Replication 영향이 검토되었습니다.
- GAS 영향이 검토되었습니다.
- 필요한 에디터 설정 또는 에셋 설정이 문서화되었습니다.
- 검증한 내용과 검증하지 못한 내용이 구분되어 있습니다.
- 남은 TODO가 정리되었습니다.
- 다음 작업이 명확합니다.

---

## 네트워크 기능 완료 기준

네트워크 기능은 가능하면 다음을 확인합니다.

- Listen Server에서의 동작
- Dedicated Server에서의 동작
- Client Join 동작
- Late Join 동작
- Replicated State 초기화
- RPC 호출 방향
- 서버 검증 로직
- 클라이언트 UI 반영

총기 파츠 또는 전투 기능은 추가로 다음을 확인합니다.

- 파츠 드랍 생성이 서버 권한으로 처리되는지
- 파츠 획득 / 장착 요청이 서버에서 검증되는지
- 무기 파츠 상태가 필요한 클라이언트에 Replication 되는지
- 발사 / 피격 / 데미지 결과가 서버 권한으로 확정되는지

---

## GAS 기능 완료 기준

GAS 기능은 가능하면 다음을 확인합니다.

- Ability 부여 / 제거 시점
- GameplayEffect 적용 / 제거 시점
- GameplayTag 추가 / 제거 시점
- Attribute 변화
- 서버 권한 처리 여부
- 예측 또는 Replication 관련 영향
- 총기 파츠 시스템과의 연결 여부

---

## 문서 완료 기준

문서 작업은 다음을 확인합니다.

- 목적이 명확합니다.
- 현재 프로젝트 범위와 맞습니다.
- 중복 내용이 과하지 않습니다.
- AGENTS.md 또는 관련 문서에서 참조 경로가 맞습니다.
- 이후 Codex가 실제 작업에 사용할 수 있는 형태입니다.
