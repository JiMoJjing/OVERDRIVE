## Implementation Summary

- 플레이어 캐릭터는 단순 인간형 캐릭터로 고정하고, 핵심 커스터마이징은 총기 파츠 시스템에 집중하도록 문서 기준을 변경했다.
- 로비 커스터마이징은 제거하고, 로비는 접속 / Ready / 게임 시작 흐름을 보여주는 시스템으로 축소했다.
- 게임 중 적 처치로 총기 파츠가 드랍되고, 서버 검증 후 즉시 장착 / 교체되는 구조를 초기 목표로 정의했다.
- 총기 파츠 슬롯은 `Barrel`, `Magazine`, `Stock`, `Scope / Sight`, `Muzzle`, `Core Module`, `Optional Attachment` 기준으로 정리했다.
- 초기 구현에서는 복잡한 인벤토리보다 즉시 교체형 파츠 시스템을 우선하도록 범위를 제한했다.

## Networking Impact

- 서버는 파츠 드랍 생성, 파츠 획득 검증, 파츠 장착 / 교체 검증, 발사 판정, 피격 / 데미지 처리를 담당한다.
- 클라이언트는 입력, UI, 로컬 시각 피드백, Hand IK, Replicated 상태 기반 화면 갱신을 담당한다.
- Late Join을 고려해 현재 장착된 파츠 ID와 슬롯별 장착 상태를 Replicated State로 관리하는 방향을 명시했다.
- 지속 상태 동기화는 Multicast보다 Replicated State를 우선하도록 유지했다.

## GAS Impact

- GAS 연동 중심을 캐릭터 / 메카 파츠에서 총기 파츠로 변경했다.
- 총기 파츠 장착 시 `GameplayAbility` 부여 / 제거, `GameplayEffect` 적용 / 제거, `GameplayTag` 갱신이 서버 권한에서 처리되도록 규칙을 정리했다.
- 무기 관련 Attribute 예시로 데미지 배율, 발사 속도 배율, 재장전 속도 배율, 반동 배율 등을 추가했다.
- `Weapon.Part.*`, `Weapon.Status.*`, `Damage.*` 계열 GameplayTag 예시를 추가했다.

## Decisions / Solutions

- `Docs/mech_equipment_rules.md`는 제거하고 `Docs/weapon_parts_rules.md`를 새 기준 문서로 추가했다.
- Mech Constructor는 메인 플레이어 에셋 후보에서 제외하고, 필요 시 적 / 보스 / 테스트용 에셋으로만 활용할 수 있다고 정리했다.
