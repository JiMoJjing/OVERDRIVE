# Project Overview

## 프로젝트 이름

OVERDRIVE

---

## 프로젝트 목적

이 프로젝트는 취업 포트폴리오용 Unreal Engine 게임 프로젝트입니다.

상업 출시나 인디게임 완성도보다, 게임 클라이언트 프로그래머로서의 기술 역량 증명이 우선입니다.

핵심 목표는 다음입니다.

- Unreal Engine C++ 기반 개발 역량 증명
- GAS 활용 능력 증명
- Dedicated Server 기반 멀티플레이 구조 경험
- 서버 권한 기반 전투와 장비 상태 처리 경험
- Modular Weapon Parts System 설계/구현 능력 증명
- AI 활용 개발 파이프라인 구축
- 가능하면 Unreal Insights 등을 활용한 최적화 경험 확보

---

## 엔진 / 기술 스택

- Engine: Unreal Engine 5.7
- Language: C++

  - Unreal Insights 활용 가능성 고려
  - Replication 최적화

  - 이펙트 / 프레임 최적화

## 게임 장르 / 구조

장르는 다음 방향입니다.

**Dedicated Server 기반 Co-op PvE Mission TPS**

플레이어는 단순 인간형 캐릭터로 미션에 진입하고, 적을 처치하며 획득한 총기 파츠를 장착해 무기를 강화한 뒤, 최종적으로 거대 보스를 처치하고 결과 화면으로 이동합니다.

---

## 핵심 게임 루프

```text

→ 무기 성능 변화

→ 결과 화면
```

## 메인 플레이어 / 에셋 방향

메인 플레이어는 단순 인간형 캐릭터를 사용합니다.

선택 이유:

- 캐릭터 외형 제작과 애니메이션 부담을 줄입니다.
- Unreal 기본 Manny 또는 유사 인간형 캐릭터와 리타겟팅하기 쉽습니다.
- 플레이어 캐릭터보다 총기 파츠 시스템, GAS, 네트워크 구조가 돋보입니다.
- Hand IK, 무기 소켓, 조준, 발사 위치 검증에 집중하기 좋습니다.

총기 에셋은 파츠화된 총기 에셋을 우선 사용합니다.

초기 우선순위:

1. 단순 인간형 플레이어 캐릭터
2. 기본 총기 에셋
3. 총기 파츠 에셋
4. 적 Placeholder
5. 보스 Placeholder
6. 맵 / 배경 Placeholder

기존 Mech Constructor 기반 로봇 플레이어 방향은 보류 또는 제외합니다. 필요 시 적, 보스, 기계형 오브젝트, 테스트용 에셋으로만 활용할 수 있습니다.

## 개발 시작 우선순위

첫 개발 목표는 다음입니다.

**Dedicated Server 로비 + 기본 인간형 플레이어 + 총기 파츠 드랍/장착 프로토타입**

초기 개발 우선순위:

1. UE 5.7 C++ 프로젝트 세팅
2. Dedicated Server 빌드 / 실행 구조 확인
3. 클라이언트 2개 이상 접속 확인
4. 로비 생성 / 접속 흐름 구현
5. 로비 내 플레이어 Ready 상태 구현
6. 기본 인간형 플레이어 캐릭터 적용
7. 기본 총기 발사 Ability 구현
8. 적 Placeholder 처치 구현
9. 총기 파츠 드랍 서버 생성 구현
10. 파츠 획득 서버 검증 구현
11. 총기 파츠 장착 / 교체 구현
12. 파츠 효과를 GAS GameplayEffect / GameplayAbility와 연결
13. 무기 상태 Replication
14. 전진형 전투 구간과 보스 Placeholder 연결
15. 결과 화면 구현
