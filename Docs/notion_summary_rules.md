# Notion Summary Rules

## 목적

Notion 정리는 작업 로그보다 더 읽기 쉬운 포트폴리오 정리용 문서입니다.

작업 로그가 개발 기록이라면, Notion 정리는 외부 리뷰어가 이해하기 위한 요약입니다.

---

## 권장 구조

```md
# 기능명

## 구현한 기능

- 무엇을 구현했는지 설명

## 필요한 이유

- 이 기능이 프로젝트에서 필요한 이유
- 포트폴리오에서 보여줄 수 있는 기술 역량

## 기술적 접근

- 주요 클래스
- 주요 데이터 흐름
- 구현 방식

## 서버 권한 구조

- 클라이언트가 요청하는 부분
- 서버가 검증하는 부분
- 서버가 확정하는 상태

## GAS 사용 방식

- 사용한 Ability
- 사용한 GameplayEffect
- 사용한 GameplayTag
- Attribute 변화

## Replication 구조

- Replicated 변수
- RPC
- Late Join 고려 여부

## 발생한 문제와 해결

- 문제
- 원인
- 해결 방식

## 결과

- 현재 동작 상태
- 확인한 내용

## 다음 작업

- 이어서 해야 할 일
```

---

## 작성 원칙

- 코드를 모르는 사람도 큰 흐름을 이해할 수 있어야 합니다.
- Raw Prompt를 그대로 붙여넣지 않습니다.
- Raw Code를 길게 붙여넣지 않습니다.
- 중요한 코드 일부는 설명과 함께 짧게 첨부할 수 있습니다.
- 기술적 의사결정 이유를 남깁니다.
